Provides the client-side aspect of the heritrix integration self test.

The garden webapp is the repository for the serverside of the intergration test.

The integration self test is run from the command line. Invocation makes the crawler go up against itself trawling the garden webapp. When done, the product -- arc and log files -- are analyzed by code herein to verify test pass or fail.

The integration self test is the aggregation of multiple individual tests each testing a particular crawler aspect. For example, the Robots test validates the crawler's parse of robots.txt. Each test comprises a directory under the garden webapp named for the test into which we put the server pages that express the scenario to test, and a class that subclasses org.junit.TestCase from this package named for the webapp directory w/ a SelfTest suffix. This class verifies test completion.

The first crop of self tests are derived from tests developed by Parker Thompson &rt;pt at . See Tests.
